# 服装・衣替えアプリ 要件定義書

## 1. 概要・目的

現在の気温・天気に応じて、当日の服装の厚さ指標と衣替え時期を提示するアプリ。

対象は性質の異なる2機能で、両者を同等に扱う。

- 当日の服装（連続・毎日利用）：その日の予報から服装の厚さを提示する
- 衣替え時期（離散・季節イベント）：気温推移からクローゼットの入替え時期を判定する

## 2. スコープ

### MVP範囲
- 地点の気温・天気・風速・湿度の取得
- 当日服装の提示（昼／朝晩の二段表示）
- 直近気温との相対提示（「一枚多め／少なめ」）
- 衣替え判定（週間平均気温の移動平均、ヒステリシス付き）
- 寒がり／暑がりの個人オフセット設定

### 対象外（将来フェーズ）
- 衣替え提案のプッシュ通知
- 手持ち衣類の登録・コーディネート提案
- 家族・複数ユーザー共有

## 3. 前提・制約

- 技術構成は Next.js + Firebase（既存構成を踏襲）。
- 相対提示・衣替え判定・順応が日次気温の時系列履歴に依存するため、履歴保存を初期から設計する。
- 衣替え判定に7日分の予報が必要。データソースは7日以上の予報を提供するものを選定する。

## 4. 機能要件

| ID | 機能 | 内容 |
|----|------|------|
| F-01 | 気象データ取得 | 地点の気温（最高／最低）・天気・風速・湿度・7日予報を取得しキャッシュする |
| F-02 | 体感温度補正 | 風速・湿度・日射・雨から補正気温を算出する |
| F-03 | 当日服装提示 | 補正気温を服装レベルに変換し、昼／朝晩の二段で表示する |
| F-04 | 相対提示 | 直近平均との差分から「一枚多め／少なめ」を提示する |
| F-05 | 衣替え判定 | 週間平均気温の移動平均で衣替え時期を判定・提案する |
| F-06 | 個人オフセット | 寒がり／暑がりのオフセットを服装レベルに反映する |
| F-07 | 気温履歴保存 | 日次の気温・湿度・予報を蓄積する |

## 5. ロジック仕様

### 5.1 体感温度補正

気温を風速・湿度・日射・雨で補正し、補正気温を得てから服装レベルに変換する。

- 風速：Missenard 等の体感式で風による低下を反映する（風速が大きいほど体感気温を下げる）
- 湿度：高湿度時は夏の暑さを増幅、冬の寒さを増幅する方向で補正する
- 雨：防水の要否フラグとして扱い、服装アドバイスに付加する
- 日射：晴天時は日中の体感を上げる方向で補正する

補正式は初期実装では風速中心とし、他要素は段階的に追加してよい。

### 5.2 服装レベルマスタ

補正気温を段階レベルに変換する。マスタは Firestore に持ち、後から調整可能とする。初期値は次のとおり。

| レベル | 気温帯 | 服装例 |
|--------|--------|--------|
| 1 | 25℃以上 | 半袖 |
| 2 | 22〜24℃ | 長袖1枚 |
| 3 | 18〜21℃ | 重ね着（カーディガン等） |
| 4 | 15〜17℃ | 薄手の上着 |
| 5 | 12〜14℃ | コート |
| 6 | 8〜11℃ | 厚手コート |
| 7 | 8℃未満 | 防寒（マフラー・手袋等） |

### 5.3 当日服装（二段表示）

日較差に対応するため、昼と朝晩を分けて表示する。

- 昼：最高気温を補正した値のレベル
- 朝晩：最低気温を補正した値のレベル

### 5.4 相対提示（順応の反映）

同じ気温でも直近の気候により体感が変わる（順応）。直近平均を基準線とし、当日との差分でアドバイスを付す。

- 基準線：直近3日間の平均気温
- 当日の補正気温が基準線より一定以上高い／低い場合に「一枚少なめ／多め」を提示する

絶対気温ベースの服装レベル（5.3）に、相対差分の補正層（5.4）を重ねる二層構成とする。

### 5.5 衣替え判定

単日でなく週間平均気温の移動平均で判定し、頻繁な切替提案を防ぐためヒステリシスを設ける。

- 判定値：直近7日の平均気温の移動平均
- 秋→冬方向：移動平均が下限閾値を継続して下回ったら「厚手への入替え」を提案
- 春→夏方向：移動平均が上限閾値を継続して上回ったら「薄手への入替え」を提案
- ヒステリシス：上げ局面と下げ局面で閾値を変え、境界付近での往復提案を防ぐ

閾値の初期値は運用で調整する（未決事項 D-01）。

### 5.6 個人オフセット

寒がり／暑がりのオフセットを服装レベルに反映する（例：寒がりはレベルを1段上げる方向）。

## 6. データモデル（Firestore）

### users/{uid}/settings
- location：地点
- offset：個人オフセット（初期値 0）
- outing_hours：外出時間帯（任意、朝晩判定の基準に使用）

### clothing_levels/{level}（マスタ）
- temp_min / temp_max：気温帯
- label：服装例

### weather_history/{location}/{date}
- temp_max / temp_min：最高／最低気温
- humidity / wind / weather：湿度・風速・天気
- forecast_7d：7日予報（衣替え判定用）

### users/{uid}/changeover_state
- current_season_mode：現在の衣替え状態（例：夏物／冬物）
- last_suggested_at：直近の提案日時

## 7. 外部データ（気象API）

全機能が同一ソースに依存するため、抽象化レイヤを挟みソースを差し替え可能にする。衣替え判定と相対提示が7日予報と履歴を要求するため、ソースは7日以上の予報を無料枠で提供する必要がある。取得結果は Firestore にキャッシュし、1日数回更新する。

候補（未決事項 D-02）：

| 候補 | 長所 | 留意点 |
|------|------|--------|
| 気象庁 | 無料・APIキー不要・日本の信頼性が高い・週間予報あり | エンドポイントが非公式・地点が区域コード単位 |
| OpenWeatherMap 等 | 緯度経度指定・7〜8日予報・体感/風/湿度を一括取得 | APIキー必要・日本の地点粒度と無料枠を要確認 |

必要データ：最高／最低気温、天気、風速、湿度、7日予報。無料枠・エンドポイント・地点粒度は実装時に確認する。

## 8. 画面構成

- 今日の服装：昼／朝晩のレベルと服装例、相対アドバイス、雨天フラグ
- 衣替え：判定状態と提案（該当時にバナー表示）
- 設定：地点、寒がり／暑がりオフセット、外出時間帯

## 9. 非機能要件

- オフライン時はキャッシュした直近データで表示する
- 服装レベルマスタと衣替え閾値はコードまたは設定で調整可能にする
- 気温履歴は最低でも数週間分を保持し、移動平均と順応判定に用いる

## 11. 補足：エアコンアプリとの共通基盤

体感温度・不快指数の計算、地点特定、寒がり／暑がりオフセット、気象データ取得・キャッシュは、エアコン設定温度アプリと共通化できる。将来的に単一アプリへ統合する場合は、これらを共有モジュールとして切り出す。
